# 监控系统使用指南

## 📊 查看监控系统状态

### 1. 查看运行中的进程
```powershell
# 查看所有Node.js进程
tasklist /FI "IMAGENAME eq node.exe" /FO TABLE

# 查看详细信息
wmic process where "name='node.exe'" get processid,commandline,workingsetsize
```

### 2. 查看监控日志
监控系统会在以下位置生成日志：
- **控制台输出**：直接显示在启动监控的终端窗口
- **日志文件**：`logs/monitor-YYYY-MM-DD.log`

### 3. 检查监控状态
运行状态检查脚本：
```bash
node check-monitor.js
```

## 📝 日志查看方法

### 实时查看日志
```powershell
# 如果有日志文件，使用tail类似工具
Get-Content logs/monitor-2026-04-01.log -Wait -Tail 10

# 或者简单查看
Get-Content logs/monitor-2026-04-01.log
```

### 查看监控数据
监控系统会记录：
- 文件变动次数
- 监控的文件数量
- 启动时间和运行时长
- 错误和警告信息

## 🔄 更新监控系统

### 方法1：重启监控（推荐）
```powershell
# 1. 停止现有监控（找到PID并结束）
taskkill /PID <monitor-pid> /F

# 2. 重新启动
node monitor-start.js
```

### 方法2：后台运行
```powershell
# 使用后台模式启动
node monitor-start.js &

# 或者使用nohup（类Unix系统）
nohup node monitor-start.js > monitor.log 2>&1 &
```

### 方法3：使用PM2（生产环境）
```bash
# 安装PM2
npm install -g pm2

# 启动监控
pm2 start monitor-start.js --name "ios-monitor"

# 查看状态
pm2 status

# 查看日志
pm2 logs ios-monitor

# 重启
pm2 restart ios-monitor

# 停止
pm2 stop ios-monitor
```

## 🚨 故障排除

### 监控停止运行
```powershell
# 1. 检查进程是否还在
tasklist | findstr node

# 2. 如果没有，重新启动
node monitor-start.js

# 3. 检查错误日志
node monitor-logs.js
```

### 监控不响应文件变动
1. 检查监控路径是否正确
2. 检查文件权限
3. 重启监控服务

### 内存占用过高
```powershell
# 查看内存使用
tasklist /FI "IMAGENAME eq node.exe" /FO TABLE

# 如果过高，重启监控
taskkill /PID <pid> /F
node monitor-start.js
```
